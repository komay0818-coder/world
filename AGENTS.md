# Development workflow

- Perform all UI changes, feature work, and bug fixes on the `dev` branch.
- Never commit or push directly to `main`.
- Keep each change set focused on 2–4 related items.
- Commit completed, validated work to `dev`.
- Do not merge `dev` into `main` without the user's explicit approval after testing.
- Treat `main` as the stable public release branch.
- Do not refactor code unrelated to the current request.
- Use the dev preview for user testing; the GitHub Pages URL remains the production site.
